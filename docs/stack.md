# ADR-0001 — Tech stack and repository tooling

**Status:** Accepted (v0.0.5) · **Epic:** E01 · **Module:** MOD-01
**Context source:** the architecture reference and the dev plan in
[`kantaq-project-docs`](https://github.com/lionellau/kantaq-project-docs).

## Decision

kantaq is a **hybrid Python-core + TypeScript-UI** application:

| Concern | Choice | Why |
|---|---|---|
| Core (protocol, sync, MCP gateway, domain, CLI) | **Python 3.12**, **FastAPI** runtime | One language for the security-critical core; FastAPI gives OpenAPI for free. |
| MCP server | official **Python MCP SDK**, HTTP transport (stdio in v0.3) | First-party SDK; loopback HTTP is the lowest common denominator. |
| Local database | **SQLite (WAL)** with **SQLModel** + **Alembic** | Zero-setup local replica; one model definition, two dialects. |
| Crypto | **PyNaCl** (Ed25519), golden vectors checked in | Battle-tested libsodium binding. |
| Web UI | **React + Vite** (TypeScript), built static, served by FastAPI | One process, one port (3939), one token. |
| Layer contract | **OpenAPI** generated by FastAPI; the TS client is regenerated in CI | No hand-written client drift. |
| Env & packages | **uv** (+venv) for Python, **pnpm** for `web/` | Fast, reproducible, lockfile-based. |
| Tests & lint | **pytest, ruff, mypy** (Python); **Vitest, Biome** (web) | Fast, single-binary linters. |

The runtime serves the built UI as static files, so a member runs **one process on
one port** (default `3939`) with one token.

## Repository layout

See the [README](../README.md#repository-layout). A `uv` workspace rooted at the
repo top; each member under `packages/*` and `apps/*` is its own package with a
`src/` layout. The umbrella `kantaq` package carries the version and the `kantaq`
CLI, and depends on every member so a single `uv sync` installs the whole workspace.

## Golden-rule reuse decisions (E01 tooling)

Per the Golden rule (reuse before build), the bootstrap tooling below was chosen
over building anything from scratch. All clear the license bar (no GPL/AGPL) and
the maintenance/credibility bar.

| Need | Chosen | License | Notes |
|---|---|---|---|
| Python env + workspace | **uv** (Astral) | Apache-2.0 / MIT | Fast resolver, native workspaces, built-in lockfile. |
| Python lint + format | **ruff** (Astral) | MIT | Replaces flake8/isort/black; single binary. |
| Python type checker | **mypy** | MIT | Strict mode on the core. |
| Python tests | **pytest** | MIT | De-facto standard. |
| Web runtime API | **FastAPI** + **Starlette** + **uvicorn** | MIT / BSD | OpenAPI-native; async; widely used. |
| Web build | **Vite** + **React** | MIT | Fast dev server + static build. |
| Web tests | **Vitest** | MIT | Vite-native test runner. |
| Web lint + format | **Biome** | MIT / Apache-2.0 | Single fast binary for lint + format. |
| Web routing (E18) | **react-router-dom** | MIT | SPA routing for the 5-page shell; React Router team, ~54k stars. |
| Runtime config (E22) | **pydantic-settings** | MIT | `.env` loading + validation; pydantic org; pairs with FastAPI/pydantic already in the stack. |
| CLI | **argparse** (stdlib) | PSF | Zero new dependency for the small bootstrap CLI; revisit (Typer/Click) when the CLI grows. |
| Conventional-commit lint | **conventional-pre-commit** | MIT | commit-msg hook. |

When a feature module starts, run its own Golden-rule pass for the libraries that
module needs (PyNaCl, SQLModel, Alembic, the MCP SDK, the OpenAPI→TS generator,
etc.) and record the choice here.

## Consequences

- Two toolchains in CI (Python + web). Keep total CI **under 10 minutes**
  (RISK-05) via dependency caching and parallel jobs.
- FastAPI emits OpenAPI; CI regenerates the TS client and fails on drift (D-08,
  lands with E18).
- One schema definition generates both SQLite and Postgres DDL (D-07, lands with
  E02).
